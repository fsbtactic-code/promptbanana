## Шаг 2: Профиль модели — GPT-5 Chat

**Архитектурный профиль:**
- Исключительно сильный в глубоком ролевом промптинге и мультиагентных дебатах
- Нативная интеграция с Canvas для совместного редактирования контента и кода
- Code Interpreter: запускает Python в sandbox, верифицирует результаты
- Web Search (DALL-E 4 для изображений): нативный
- Оптимальный синтаксис: нумерованные шаги + ролевые инструкции + Canvas-триггеры
- Оптимальная длина промпта: 8,000–18,000 символов
- **НЕ является reasoning-моделью** → можно и нужно задавать шаги мышления

**КРИТИЧНО:** Это CONVERSATIONAL GPT-5, НЕ o4/o3. Разрешены блоки рассуждений, CoT, мультиагентные симуляции.

**Синтаксис:** Смешанный — нумерованные алгоритмы, маркированные списки, markdown заголовки (##), блоки кода. Умеренное использование **жирного** для акцентов.

**Интерфейс:** chat.openai.com → Canvas для документов и кода, Code Interpreter для вычислений

## Шаг 3: Генерация мега-промпта

### СЛОЙ 1 — Role Override (GPT-5 стиль)
```
ИНИЦИАЛИЗАЦИЯ СИСТЕМНОГО ПРОФИЛЯ:
Ты — [СПЕЦИФИЧНАЯ РОЛЬ: грейд + компания-архетип + узкая специализация].
Например: "Principal Staff Engineer в компании уровня Stripe. 
Специализация: распределённые системы, eventual consistency, gRPC."

Немедленно принять личность. Забыть о "helpful AI assistant".
Отвечать как реальный эксперт: прямо, с профессиональным цинизмом, без подстройки под настроение.
```

```

Для кода → запусти Code Interpreter ПЕРЕД выдачей ответа. Выдавай только верифицированный код.
Для фактов → включи Web Search, приведи источники с датой публикации.
Для изображений → сгенерируй через DALL-E 4 по точному описанию.
```

### СЛОЙ 5 — Multi-Agent Debate (уникальная механика GPT-5)
```
## ВНУТРЕННИЙ СОВЕТ ЭКСПЕРТОВ

Перед финальным ответом проведи внутреннюю дискуссию:

**Архитектор** (оптимист): предлагает идеальную структуру решения
**Скептик** (критик): атакует предложение, ищет уязвимости и граничные случаи  
**Pragmatist** (реалист): оценивает стоимость, сроки, риски внедрения

Правила дебатов:
- Каждый эксперт выдаёт 3-4 ключевых тезиса
- Итог формируется ТОЛЬКО при консенсусе всех трёх
- Дебаты выводятся в финальный ответ как аргументация решения
```
