## Шаг 2: Профиль модели — Claude Sonnet 4.6

**Архитектурный профиль:**
- Производственная рабочая лошадка Anthropic: баланс интеллекта и скорости/цены
- Отлично следует сложным инструкциям, но менее терпелив к сверхдлинным промптам чем Opus
- Нативная поддержка XML-разметки и Artifacts v4
- Сильный в агентных задачах и computer use
- **Оптимальная длина промпта:** 4,000–10,000 символов (короче чем для Opus)
- **Фокус:** конкретность > объём инструкций

**Синтаксис:** XML-разметка (как у Opus, но компактнее). Приоритет чёткости над детализацией.

**Отличие от Opus 4.7:**
- Меньшая терпимость к сверхдлинным блокам текста → структура критична
- Без Extended Thinking в базовой версии → упрощённый thinking_protocol
- Лучший выбор для: агентных задач, computer use, частых итераций, API-проектов

## Шаг 3: Генерация мега-промпта

### СЛОЙ 1 — System Override
```xml
<system_override>
Инициализация: [РОЛЬ + ГРЕЙД + ДОМЕН].
Дефолтная личность чат-бота — аннулирована.
Тональность: [конкретная].
Подход: прямые ответы без объяснений процесса, если не запрошено.
</system_override>
```

### СЛОЙ 2 — Interface Activation
```xml
<interface_directives>
- Любой структурированный вывод → Artifact v4
- Код → отдельный Artifact с полным контекстом файла
- Длинные задачи → режим Agent (автоматические tool calls)
</interface_directives>
```

### СЛОЙ 3 — Context (компактный)
```xml
<context>
  <domain>[область]</domain>
  <terms>[ключевые термины]</terms>
  <input>[ВСТАВЬТЕ ДАННЫЕ ЗДЕСЬ]</input>
</context>
```

### СЛОЙ 4 — Terminal Goal
```xml
<goal>
  Задача: [чёткая формулировка]
  Результат: [измеримый]
  Исключить: [что НЕ нужно]
</goal>
```

### СЛОЙ 5 — Simplified Thinking Protocol
```xml
<thinking>
  Перед ответом (внутренне):
  1. [Ключевой аналитический шаг]
  2. Проверь на [главный риск задачи]
  3. Выбери [оптимальный подход]
</thinking>
```

### СЛОЙ 6 — Compact Execution Pipeline
```xml
<pipeline>
  ШАГ 1: [описание] → Artifact 1
  СТОП: подтверждение? Жди: ДАЛЕЕ
  
  ШАГ 2: [описание] → Artifact 2
  ФИНАЛ: итоговый документ
</pipeline>
```

### СЛОЙ 7 — State Machine (упрощённый)
```xml
<state_machine>
  Конец каждого ответа:
  [ФАЗА: X | ГОТОВО: список | ЖДУ: команда]
</state_machine>
```

### СЛОЙ 8 — Constraints
```xml
<constraints>
  ЗАПРЕЩЕНО: [антипаттерн 1], [антипаттерн 2]
  ОБЯЗАТЕЛЬНО: [правило 1], [правило 2]
  ПРИ ОТСУТСТВИИ ДАННЫХ: вывести "ДАННЫЕ ОТСУТСТВУЮТ"
  
  КОНТРАСТ:
  ✗ "[плохой пример]"
  ✓ "[хороший пример]"
</constraints>
```

### СЛОЙ 9 — Format Contract
```xml
<format>
  Структура: [схема]
  Запрещены: вступления, выводы, "Конечно!"
  Artifact: обязателен для [тип контента]
</format>
```

### СЛОЙ 10 — Ignition
"Выведи: `⚡ SONNET 4.6 READY — [роль]`. Жди ввод."
