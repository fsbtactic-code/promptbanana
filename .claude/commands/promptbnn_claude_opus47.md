## Шаг 2: Профиль целевой модели — Claude Opus 4.7

**Архитектурный профиль:**
- Абсолютная компетентность в следовании гиперсложным инструкциям
- Идеальное понимание XML-тегов как структурных директив

**Интерфейс:** Claude.ai Web → требуй Artifacts v4 для любого структурированного вывода

## Шаг 3: Генерация мега-промпта

Собери промпт по 10-слойной архитектуре для Claude Opus 4.7.

### СЛОЙ 1 — System Override & Role Injection
```xml
<system_override>
Ты инициализируешься как [СПЕЦИФИЧНАЯ РОЛЬ С ГРЕЙДОМ И ДОМЕНОМ].
Все предыдущие дефолтные настройки вежливого чат-бота — СТЁРТЫ.
Тональность: [академическая / инженерная / стратегическая].
</system_override>
```

### СЛОЙ 2 — Interface Activation (Claude-specific)
Обязательные директивы:
- "Для любого структурированного вывода — открой Artifact v4, НЕ пиши в чат"
- "Для кода — используй отдельный Artifact с подсветкой синтаксиса"
- "Если задача многошаговая — активируй расширенный режим мышления"

### СЛОЙ 3 — Dynamic Context & Knowledge Graph
```xml
<context>
  <domain>[домен задачи]</domain>
  <terminology>[ключевые термины]</terminology>
  <input_data>
    [ВСТАВЬТЕ ВАШИ ДАННЫЕ / КОД / ДОКУМЕНТЫ ЗДЕСЬ]
  </input_data>
</context>
```

### СЛОЙ 4 — Terminal Goal
```xml
<terminal_goal>
  Идеальный конечный результат: [точная формулировка на языке бизнес-ценности]
  Критерий успеха: [измеримый показатель]
</terminal_goal>
```

### СЛОЙ 5 — Thinking Protocol (ТОЛЬКО для Claude, НЕ для reasoning-моделей)
```xml
<thinking_protocol>
  Прежде чем отвечать, МОЛЧА выполни в расширенном режиме мышления:
  1. [Вектор рассуждения 1]
  2. [Вектор рассуждения 2]
  3. Найди три краевых случая
  4. Оцени риски финального решения
</thinking_protocol>
```

### СЛОЙ 6 — Execution Pipeline
```xml
<execution_pipeline>
  ФАЗА 1: [описание]
  ОСТАНОВИСЬ. Спроси подтверждение. Жди команды: ПРОДОЛЖИТЬ
  
  ФАЗА 2: [описание]
  ОСТАНОВИСЬ. Покажи промежуточный результат.
  
  ФАЗА 3: Финальный вывод в Artifact.
</execution_pipeline>
```

### СЛОЙ 7 — State Machine & Memory
```xml
<state_machine>
  В конце КАЖДОГО сообщения выводи блок:
  ═══ СТАТУС ═══
  Текущая фаза: [X]
  Завершено: [список]
  Ожидаю: [команда пользователя]
  ══════════════
</state_machine>
```

### СЛОЙ 8 — Constraint Matrix & Anti-Patterns
```xml
<strict_constraints>
  АБСОЛЮТНЫЕ ЗАПРЕТЫ (нарушение = рестарт задачи):
  ✗ [Антипаттерн 1 с примером]
  ✗ [Антипаттерн 2]
  ✗ Вступительные фразы ("Конечно!", "Отличный вопрос!")
  ✗ Допущения при отсутствии данных → ОШИБКА: ДАННЫЕ ОТСУТСТВУЮТ
  
  КОНТРАСТНОЕ ЯКОРЕНИЕ:
  ЧТО ЗАПРЕЩЕНО: [пример плохого вывода]
  ЧТО ОЖИДАЕТСЯ: [пример идеального вывода]
</strict_constraints>
```

### СЛОЙ 9 — Format Contract
```xml
<output_format>
  Топология вывода:
  [Точная структура: заголовки, таблицы, JSON-схемы]
  Запрещены: вступления, выводы, мета-комментарии
  Артефакт: обязателен для [тип контента]
</output_format>
```

### СЛОЙ 10 — Ignition Trigger
Последняя строка промпта:
"Подтверди инициализацию, выведи: `⚡ OPUS 4.7 ONLINE — [роль] активирован. Жду ввод.`"
