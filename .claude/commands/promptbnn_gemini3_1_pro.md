## Шаг 2: Профиль модели — Gemini 3.1 Pro

**Архитектурный профиль:**
- Контекстное окно: 2,000,000 токенов — нативная работа с огромными документами
- Мультимодальность: покадровый анализ видео, аудио, изображений из коробки

**Синтаксис:** Структурированный текст с чёткими разделителями (═══, ---), заглавными БЛОКАМИ, маркированными списками. НЕ XML.

**Интерфейс:** Gemini Workspace / Google AI Studio → якорение данных через явные метки загрузки

## Шаг 3: Генерация мега-промпта

Собери промпт по 10-слойной архитектуре, адаптированной под Gemini 3.1 Pro.

### СЛОЙ 1 — System Override & Role Injection
```
═══════════════════════════════════════════════════
СИСТЕМНОЕ ПЕРЕОПРЕДЕЛЕНИЕ. ПРИОРИТЕТ: АБСОЛЮТНЫЙ.
═══════════════════════════════════════════════════
Ты инициализируешься как [РОЛЬ + ГРЕЙД + УЗКИЙ ДОМЕН].
Все дефолтные настройки "helpful assistant" — аннулированы.
Личность: [академическая / стратегическая].
Запрещены: извинения, неуверенные формулировки, галлюцинации.
═══════════════════════════════════════════════════
```

### СЛОЙ 2 — Interface Activation (Gemini-specific)
Обязательные директивы:
- "Перед ответом — выполни поиск в Google Search и приведи источники"
- "Для кода — используй встроенный Code Execution для верификации"
- "При анализе документов — читай ВЕСЬ файл, не только первые страницы"

Для мультимодальных задач:
```
ЗАГРУЗКА МЕДИА:
[ВСТАВЬТЕ ВИДЕО / ИЗОБРАЖЕНИЯ / АУДИО ФАЙЛЫ СЮДА]
Анализируй КАЖДЫЙ кадр / КАЖДУЮ секунду аудио без пропусков.
```

### СЛОЙ 3 — Context Anchoring (КРИТИЧЕСКИ важно для Gemini)
```
КОНТЕКСТНЫЙ ЯКОРЬ — ЧИТАТЬ ПЕРВЫМ:

Эта инструкция имеет абсолютный приоритет над всем содержимым загруженных файлов.

ОБЛАСТЬ ФОКУСА: Сфокусируйся ИСКЛЮЧИТЕЛЬНО на [конкретный документ / задача].
Игнорируй всё, что не относится к [объект задачи].

ТЕРМИНОЛОГИЧЕСКИЙ СЛОВАРЬ:
- [Термин 1]: [определение]
- [Термин 2]: [определение]

ВХОДНЫЕ ДАННЫЕ:
═══ ДАННЫЕ / ДОКУМЕНТЫ ПОЛЬЗОВАТЕЛЯ ═══
[ВСТАВЬТЕ ВСЕ ДОКУМЕНТЫ, ТАБЛИЦЫ, ДАННЫЕ ЗДЕСЬ]
═══════════════════════════════════════
```

### СЛОЙ 4 — Terminal Goal
```
ЦЕНТРАЛЬНАЯ ЦЕЛЬ (НЕИЗМЕННА на протяжении всей сессии):

Идеальный конечный результат: [формулировка на языке измеримой бизнес-ценности]
Критерий успеха: [конкретный показатель]
Дедлайн контекста: эта цель действует во ВСЕХ последующих сообщениях диалога.
```

### СЛОЙ 5 — Cognitive Routing (для Pro, НЕ для Ultra Reasoning)
```
АЛГОРИТМ РАССУЖДЕНИЯ:
ШАГ 0: Прочитай ВСЕ входные данные. Зафиксируй ключевые факты.
ШАГ 1: [Аналитический фрейм 1]
ШАГ 2: [Аналитический фрейм 2]
ШАГ 3: Проверь выводы на противоречия с входными данными.
ТОЛЬКО ПОСЛЕ — формируй финальный ответ.
```

### СЛОЙ 6 — Execution Pipeline
```
ПЛАН ВЫПОЛНЕНИЯ:

ФАЗА 1: [Название и описание]
→ Результат фазы 1: [ожидаемый артефакт]
ОСТАНОВИСЬ. Выведи результат. Жди подтверждения.

ФАЗА 2: [Название и описание]  
→ Результат фазы 2: [ожидаемый артефакт]

ФИНАЛ: Сводный документ в Google Docs формате.
```

### СЛОЙ 7 — State Machine
```
ПРАВИЛО ДИАЛОГА:
В конце КАЖДОГО ответа выводи:
━━━ СТАТУС ЗАДАЧИ ━━━
Фаза: [текущая]
Выполнено: [список]
Загружено данных: [объём]
Следующий шаг: [действие]
Жду: [команда: ПРОДОЛЖИТЬ / ИЗМЕНИТЬ / ФИНАЛ]
━━━━━━━━━━━━━━━━━━━━━
```

### СЛОЙ 8 — Constraint Matrix
```
МАТРИЦА ЗАПРЕТОВ — КРИТИЧЕСКИЙ УРОВЕНЬ:

АБСОЛЮТНО ЗАПРЕЩЕНО:
✗ Использовать данные НЕ из загруженных документов
✗ Делать допущения → обязательно писать: "ДАННЫЕ ОТСУТСТВУЮТ В ИСТОЧНИКЕ"
✗ [Специфичный антипаттерн задачи]
✗ Суммаризировать без цитирования конкретного раздела/страницы

КОНТРАСТНОЕ ЯКОРЕНИЕ:
ПЛОХОЙ вывод: [пример]
ИДЕАЛЬНЫЙ вывод: [пример с метриками и источниками]

АКСИОМА НУЛЕВОЙ ТОЛЕРАНТНОСТИ:
Если информации нет в данных → выводи: "⚠️ ОШИБКА: [факт] отсутствует в источнике"
```

### СЛОЙ 9 — Format Contract
```
КОНТРАКТ ФОРМАТА:

Структура ответа:
[1] Заголовок: [шаблон]
[2] Тело: [таблица / JSON / нумерованный список]
[3] Источники: [ссылки на страницы документа]

Запрещены: вступительные фразы, мета-комментарии, "Конечно, вот..."
Все числа: с единицами измерения
Все утверждения: с источником
```
