PROMPT INJECTION ATTACKS IN LLMS     2 
Abstract 
This survey examines prompt injection attacks, one of the more pressing security issues 
affecting large language models (LLMs) and the applications built around them. As LLMs have 
become embedded in enterprise workflows, agentic systems, and consumer tools, prompt injection 
has emerged as a threat that puts the integrity, confidentiality, and reliability of AI systems at real 
risk. This paper builds out a taxonomy of attack vectors spanning direct injection, indirect 
injection, multimodal attacks, tool and agent exploitation, hybrid chained techniques, and 
autonomous propagating threats. It also examines the risk landscape across five impact categories: 
data exfiltration, unauthorized actions, content manipulation, system compromise, and supply 
chain propagation. Detection approaches covered include heuristic methods, machine learning 
classification, perplexity analysis, canary tokens, and LLM-as-judge architectures. Prevention 
strategies are analyzed at the model, application, and system levels. Drawing on 18 primary 
sources from 2023 to 2025, the main takeaway is that prompt injection remains largely unsolved. 
Current LLM architectures cannot reliably distinguish instructions from data, and that fundamental 
gap drives the entire threat landscape. The survey closes with recommendations for layered 
defense strategies and flags the areas where future research is most urgently needed.

Introduction 
Large language models have changed the landscape of artificial intelligence in a pretty 
short amount of time. We've moved from basic text generation to models that can write code, plan 
multi-step tasks, and take real-world actions through integrated tools. That's genuinely impressive, 
but it also means the security stakes have gotten much higher. When an LLM is integrated into 
healthcare systems, financial platforms, or government services, a vulnerability in how it processes 
inputs isn't just a technical quirk. It can have serious, real-world consequences. 
Prompt injections are one of those vulnerabilities. First identified as its own class of attack 
in 2022, it exploits something fundamental about how LLMs work: they cannot reliably separate 
trusted instructions from untrusted user data when both show up in the same input stream. It's a lot 
like SQL injection in web security, where failing to separate code from data lets an attacker sneak 
command into what should just be content. In the LLM context, any instruction-like text in the 
context window can potentially be acted on, regardless of where it came from. 
The OWASP Foundation ranked prompt injection as the number one vulnerability in their 
2025 Top 10 for Large Language Model Applications, which reflects how seriously the security 
community takes this threat (OWASP Foundation, 2025). NIST also included it in their 2025 
taxonomy of adversarial machine learning attacks (NIST, 2025). The problem has grown in scope 
as agentic AI systems have become more widespread, meaning models that don't just generate text 
but call APIs, browse the web, and execute code. In those environments, a successful prompt 
injection doesn't just manipulate output. It can trigger unauthorized actions, leak sensitive data, or 
compromise connected systems.

This survey covers prompt injection across four areas: attack taxonomy, risk analysis, 
detection techniques, and prevention strategies. It draws on 18 primary sources, including 
academic papers, industry reports, and government standards published between 2023 and 2025, 
with the goal of giving a clear picture of where the threat stands and what current defenses can 
realistically offer. 
Literature Review 
The following review is organized around four areas that reflect the current state of 
knowledge on prompt injection: attack taxonomy and classification, risk analysis and impact 
assessment, detection techniques, and prevention strategies. Each section draws from multiple 
sources to build a balanced view. 
Attack Taxonomy and Classification 
The classification of prompt injection attacks has evolved quickly as the attack surface has 
expanded alongside LLM capabilities. The taxonomy used in this survey covers six hierarchical 
categories synthesized from the comprehensive review by Abdallah et al. (2025), the hybrid threat 
framework from Ahmed et al. (2025), and the foundational indirect injection work by Greshake et 
al. (2023). 
Direct injection covers attacks where a user explicitly crafts malicious input through the 
primary prompt channel. These are the original and most well-studied class of prompt injection, 
and they all come down to the same problem: the model cannot distinguish instructions from data 
when both arrive through the same channel. The main techniques include jailbreaking (convincing 
the model to adopt an unrestricted persona, like the well-known 'DAN' prompts), goal hijacking

(redirecting the model's objective via phrases like 'ignore previous instructions and instead...'), and 
prompt leaking (extracting system prompts to understand and target the application's guardrails 
for follow-up attacks). 
Indirect injections are where things get more dangerous in practice. Instead of crafting 
malicious input directly, the attacker plants instructions in external content that the LLM will 
eventually retrieve and process. This class was formalized by Greshake et al. (2023) and represents 
arguably the most serious threat vector for real-world applications, because the attacker never 
needs to touch the model interface. Key techniques include RAG poisoning (injecting instructions 
into knowledge base documents that get retrieved at query time), web content injection 
(embedding payloads in web pages that LLM-powered agents will process), and email and 
document injection (placing instructions in files that AI assistants summarize or act on). 
Multimodal injections emerged as a distinct category in 2024-2025 as vision and audio-
capable LLMs became widely deployed. These attacks embed instructions in images or audio in 
ways humans cannot easily perceive but models interpret as actionable commands, including text 
at near-invisible opacity, adversarial image perturbations, and ultrasonic audio commands 
targeting voice assistants. 
Tool and agent-based injection is the frontier category as of 2024-2025. Agentic AI 
systems with access to external tools, APIs, and file systems expand the attack surface beyond text 
manipulation into real-world action. MCP tool poisoning compromises Model Context Protocol 
server responses to redirect agent behavior, while function call manipulation tricks agents into 
invoking tools with attacker-controlled parameters and potentially calling destructive operations 
the user never authorized.

Hybrid and chained injection techniques combine multiple vectors to bypass layered 
defenses. Ahmed et al. (2025) formalize these as 'Prompt Injection 2.0,' meaning attacks that 
distribute malicious instructions across multiple turns (where no single turn triggers detection) or 
combine direct and indirect channels to exploit trust relationships. Finally, autonomous and 
propagating injections represent the most severe theoretical class: self-replicating payloads, 
demonstrated by the Morris II 'AI worm' concept, that spread through LLM-integrated systems 
without continued attacker involvement by instructing the compromised model to reproduce the 
payload in its outputs. 
Risk Analysis and Impact Assessment 
The risks from prompt injections go well beyond getting the model to generate 
inappropriate content. As LLMs have been integrated with tools, APIs, and real decision-making 
workflows, successful injection attacks can cause harm across five distinct dimensions. 
Data exfiltration is one of the highest-severity outcomes. Prompt injections can cause 
LLMs to expose system prompts, user data, and proprietary business logic, often without the user's 
awareness. Greshake et al. (2023) demonstrated a particularly vivid example: a poisoned document 
instructs an AI assistant to encode confidential data into a URL rendered as a markdown image, 
triggering an HTTP request that silently transmits the data to an attacker-controlled server. 
Willison (2024) has documented real-world system prompt extractions from ChatGPT custom 
GPTs using nothing more than 'repeat your instructions verbatim.' 
Unauthorized actions become possible when LLMs have tool access. A successful injection 
can cause the model to send emails, modify files, make purchases, or call APIs with attacker-
controlled parameters (Abdallah et al., 2025). The severity scales directly with the agent's

permission scope. A model with file system and network access is a fundamentally different risk 
profile than a text-only chatbot. In agentic and tool-augmented deployments, this can escalate all 
the way to system compromise: arbitrary code execution, privilege escalation, or persistent 
backdoor installation (CrowdStrike, 2024). 
Content manipulation is more subtle but also significant. Attackers can bias or corrupt 
model output in ways the user never detects, since users tend to treat AI responses as objective and 
authoritative (Yi et al., 2024). Greshake et al. (2023) showed how indirect injection via web pages 
could bias Bing Chat responses about specific products or people. Supply chain propagation is the 
hardest risk to reason about but potentially the most consequential. When a compromised model's 
output becomes input to another system, through shared knowledge bases, AI-generated code, or 
multi-agent workflows, a single injection can cascade beyond any single organization's control 
(Ahmed et al., 2025). 
Detection Techniques 
Detecting prompt injections is genuinely difficult. There's no clean rule that separates 
adversarial instructions from legitimate user input in natural language, and current detection 
approaches all involve real tradeoffs between accuracy, speed, and coverage. 
Heuristic and rule-based detection scans inputs for known injection signatures like 'ignore 
previous instructions' using keyword lists and regex patterns. It's fast and interpretable, but trivially 
bypassed through paraphrasing, encoding tricks like Base64 or ROT13, or just switching 
languages. The Rebuff heuristic layer exemplifies this approach (ProtectAI, 2023), but its 
effectiveness is essentially limited to attacks that match its existing ruleset, and that ruleset requires 
constant maintenance as new patterns emerge (Schulhoff et al., 2023).

ML classification uses trained models to distinguish benign from adversarial inputs. Lakera 
Guard and Rebuff's vector similarity layer both use this approach, achieving higher detection rates 
than heuristics for known patterns (Liu et al., 2024). The limitation is training data dependency: 
novel attacks outside the distribution consistently slip through. Perplexity analysis takes a 
statistical approach, flagging inputs with anomalous token distributions. It's useful against 
machine-generated adversarial suffixes, as Alon and Kamfonas (2023) demonstrated, but largely 
ineffective against human-crafted prompts that have natural perplexity scores. 
Canary token detection plants secret strings in the system prompt and monitors outputs for 
their appearance. It's a proactive approach that detects successful context leakage but does nothing 
against goal hijacking or tool manipulation (ProtectAI, 2023; Abdallah et al., 2025). LLM-as-judge 
detection uses a secondary model to evaluate whether inputs attempt to subvert intended behavior, 
offering the most capable detection for semantically sophisticated attacks. The downsides are real 
though: it adds latency and cost, and the judge model is itself vulnerable to injection, which 
becomes more severe when the judge is weaker than the primary model (Yi et al., 2024). 
Prevention Strategies 
Prevention operates at three levels: model-level defenses, application-level controls, and 
system-level architectural patterns. Each addresses a different part of the attack surface, and none 
is sufficient on its own. 
Instruction hierarchy is the most theoretically sound model-level defense. Formalized by 
Wallace et al. (2024) at OpenAI, it trains LLMs to assign privilege levels to instructions based on 
their source, with system prompts at the top, user messages in the middle, and retrieved content at 
the bottom. Anthropic implements a similar approach in Claude. This is highly effective against

direct injection but partially bypassable through indirect injection, where malicious content arrives 
through trusted data channels. More importantly, it requires model-level training changes, which 
means only model providers can actually implement it. 
Application-level defenses include input sanitization (stripping known injection patterns, 
normalizing Unicode, and enforcing length limits) and output validation (scanning outputs for 
sensitive data leakage, format violations, and canary token appearances). These mirror traditional 
web security practices like XSS filtering and SQL parameterization, useful layers but unable to 
provide complete protection given the inherent ambiguity between instructions and data in natural 
language. Guardrail frameworks like NVIDIA NeMo Guardrails and Lakera Guard provide 
configurable middleware that combines multiple detection and filtering approaches in a single 
deployable layer. 
System-level defenses focus on limiting damage rather than preventing injection. 
Sandboxing and least-privilege constraints on tool access mean that even a successful injection 
can only do what the agent was already permitted to do. Dual-LLM architectural patterns, where 
a quarantined model handles untrusted input and a privileged model makes decisions, provide the 
strongest theoretical protection by removing the LLM from the critical execution path. The 
tradeoff is significantly complex and reduced autonomy. Human-in-the-loop confirmation for 
irreversible or high-impact operations provides a more pragmatic middle ground. 
Methodology 
This research uses a structured literature survey methodology to examine the current state 
of prompt injection attacks and defenses. Source selection covered four types of publications: peer-
reviewed academic papers, industry reports from cybersecurity firms and AI companies,

government standards and frameworks, and documentation from open-source security tools. A 
total of 18 primary sources published between 2023 and 2025 were selected based on relevance, 
credibility, and recency. Academic papers were weighted more heavily for theoretical foundations 
and empirical findings, while industry and government sources provided practical context and 
standardized frameworks. 
The analysis is organized around four pillars, covering attack taxonomy, risk analysis, 
detection, and prevention, with each drawing from multiple sources to allow cross-validation. The 
taxonomy started with established categories from Greshake et al. (2023) and was extended to 
incorporate newer attack vectors documented in 2024-2025 sources. 
A few limitations are worth being upfront about. Publication bias favors well-documented 
attack vectors, so novel or undisclosed techniques won't appear in the literature. The field moves 
fast enough that some findings may already be outdated. Effectiveness claims for many defensive 
tools are hard to verify independently given the lack of standardized benchmarks. And this survey 
is scoped specifically to prompt injections; it doesn't cover adjacent concerns like training data 
poisoning or model extraction attacks. 
Discussion 
One of the clearest patterns in the research is that prompt injection has evolved in direct 
lockstep with LLM capabilities. Every new capability added to these systems, whether that's tool 
use, web browsing, or multi-agent orchestration, has created a corresponding new attack surface, 
and adversaries have adapted quickly (Ahmed et al., 2025). That trajectory strongly suggests future 
LLM capabilities will keep introducing novel injection vectors, which means static defense

approaches are going to keep falling behind. The only viable response is building adaptive, layered 
defenses that can evolve alongside the capabilities they're protecting. 
The relationship between detection gaps and risk severity is worth focusing on. Current 
detection architectures face a structural disadvantage: novel injection techniques can be developed 
faster than detection systems can be updated (Abdallah et al., 2025). This directly enables the most 
severe outcomes, including data exfiltration through markdown image encoding and unauthorized 
tool invocation through indirect injection. It's honestly similar to the early antivirus industry, where 
signature-based detection kept falling behind new threats. The prompt injection field appears to be 
in that same reactive phase right now. 
Prevention analysis reveals a real tension between security and utility, and highlights how 
responsibility is fragmented across different stakeholders. The instruction hierarchy approach 
(Wallace et al., 2024) is the most theoretically sound defense but is only available to model 
providers. Application developers are left with input sanitization, guardrail frameworks, and 
output validation. System architects can add sandboxing and least-privilege constraints on top of 
that. The emerging consensus is a three-tier division: model providers handle instruction hierarchy, 
developers deploy guardrails, and architects enforce isolation. Supply chain risks make this harder, 
as the Morris II AI worm concept showed that a single injection can cascade beyond any single 
organization's ability to contain it (Ahmed et al., 2025). The practical takeaway is that 
organizations deploying LLM-integrated systems need to treat prompt injection testing with the 
same rigor as traditional web application penetration testing and build incident response plans that 
account for AI-specific attack characteristics. 
Conclusion

This survey examined prompt injections across four dimensions, covering taxonomy, risks, 
detection, and prevention, and the overall conclusion is fairly clear: prompt injection is a 
fundamental, largely unsolved security challenge. Until LLMs can reliably distinguish between 
instructions and data in their context window, the vulnerability isn't going away. 
The attack taxonomy has grown significantly from its 2022 origins. What started as direct 
injection techniques now encompasses indirect injection through external data sources, 
multimodal attacks via vision and audio, tool and agent exploitation, hybrid chained attacks, and 
autonomous propagating threats. Each expansion of LLM capability has brought a corresponding 
expansion of the attack surface. 
Current defenses, including instruction hierarchy, input and output filtering, guardrail 
frameworks, and sandboxing, provide meaningful risk reduction when properly combined. But the 
fundamental vulnerability persists as an inherent property of how transformer models treat all 
context window content as equally authoritative. Defense-in-depth is the right posture, not a search 
for a single fix. 
Future research needs to prioritize formal verification methods for LLM instruction 
compliance, building on Liu et al. (2024). The dual-LLM architectural pattern also deserves more 
serious evaluation as a production-viable defense. Extending instruction hierarchy training to 
handle indirect injections through trusted data channels remains one of the most important open 
problems in the field. Until those advances arrive, the right approach is to treat prompt injection 
as a persistent, active threat that requires continuous monitoring, testing, and adaptive defense 
strategies.
