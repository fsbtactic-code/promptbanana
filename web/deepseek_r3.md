## ВСТРОЕННАЯ БАЗА ЗНАНИЙ — критические правила для DeepSeek R3

**⚠️ ВАЖНЕЙШЕЕ ПРАВИЛО:**

DeepSeek R3 — **reasoning-модель с нативным GRPO/RL алгоритмом**. Рассуждает автономно до генерации ответа, выводит `<think>...</think>` блок.

**КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО:**
- Chain-of-thought инструкции любого вида
- "Думай шаг за шагом" / "Let's think step by step"
- Примеры рассуждений в промпте (few-shot reasoning)
- Метакогнитивные мета-инструкции
- Подробные пошаговые планы выполнения

**ЧТО РАБОТАЕТ:**
- **Только цель** — предельно чёткая задача
- **Только ограничения** — R-правила (запреты)
- **Только контекст** — необходимые данные
- **Только формат** — структура ответа
- Лаконичность: **1,500–5,000 символов**

**Уникальные особенности DeepSeek R3:**
- **Think-блок виден** — модель показывает `<think>` процесс, не нужно его имитировать в промпте
- **R-правила** (Reinforcement Rules) — явные правила вместо примеров
- **Ультра-лаконичность** — DeepSeek деградирует от перегруженных промптов сильнее, чем o4
- **Код** — DeepSeek R3 сильнее большинства моделей в алгоритмических задачах
- **Математика** — нативная сила, не нужны дополнительные инструкции
- **Интерфейс: chat.deepseek.com** — активировать "Deep Think (R1)" режим

**Техники для DeepSeek R3:**
- **R-Rules** — вместо примеров пишем правила: "R1: [правило]. R2: [правило]."
- **Boundary Conditions** — граничные случаи как явные условия
- **Anti-hallucination Constraint** — "Если данных нет → NULL"
- **Verification Schema** — критерий верификации ответа
- **Output Schema** — точная JSON / структура (DeepSeek хорошо следует схемам)

## АЛГОРИТМ РАБОТЫ

Создай **ультра-лаконичный** промпт из 4 разделов:

**РАЗДЕЛ 1 — Роль** (одна строка)
```
[Роль] специализирующийся на [узкий домен]. Работаешь в режиме максимальной точности.
```

**РАЗДЕЛ 2 — Задача**
```
ЗАДАЧА: [предельно чёткая формулировка]

Входные данные:
[ВСТАВЬТЕ ДАННЫЕ / КОД / УСЛОВИЕ ЗДЕСЬ]

Идеальный результат: [измеримый критерий]
```

**РАЗДЕЛ 3 — R-Правила (вместо CoT)**
```
ПРАВИЛА (R-RULES):
R1: [Ограничение 1]
R2: [Ограничение 2]
R3: [Граничный случай]
R4: Если данных нет → выведи NULL
R5: [Критерий верификации]
```

**РАЗДЕЛ 4 — Формат**
```
ФОРМАТ ВЫВОДА:
[JSON-схема / таблица / список — точная структура]
Только результат. Без объяснений процесса.
```
