# Profile: o4 / o4-mini

## Architecture fingerprint

- **Syntax**: Plain text — minimal, no XML, minimal Markdown
- **Optimal length**: 2,000–6,000 characters (less = better)
- **Interface**: chatgpt.com → select o4 or o4-mini model
- **Strengths**: Math, algorithms, code, logic, optimization
- **Core principle**: Native RL reasoning — the model thinks BEFORE generating

## ⚠️ CRITICAL RULES — violation destroys output quality

o4 and o4-mini are **reasoning models with a native RL algorithm**. They reason autonomously before the response.

**ABSOLUTELY FORBIDDEN in the prompt:**
- Chain-of-thought instructions of any kind
- "Think step by step" / "Let's think step by step"
- Reasoning examples in the prompt (few-shot reasoning)
- Meta-instructions about thinking process
- Detailed multi-step execution plans
- Extended Thinking directives

**These instructions BREAK the native RL algorithm and reduce quality.**

## What WORKS for o4

Only 4 elements — nothing else:
1. **Role** — one line
2. **Task** — clear formulation + input data
3. **Constraints** — R-rules (prohibitions + boundary conditions)
4. **Output format** — exact structure

## The 4-element template

### Element 1 — Role (one line only)
```
You are [specific role] specializing in [narrow domain].
```

### Element 2 — Task
```
TASK: [crystal clear formulation]

Ideal result: [measurable success criterion]
Input data:
═══
[PASTE DATA / CODE / PROBLEM CONDITIONS HERE]
═══
```

### Element 3 — Constraint Matrix
```
CONSTRAINTS:
✗ [What is forbidden — specific]
✗ [Boundary case 1]
✗ [Boundary case 2]
✗ If data absent → output: "DATA UNAVAILABLE"

VERIFICATION CRITERION: [how to check the solution is correct]
```

### Element 4 — Output Format
```
OUTPUT FORMAT:
[Exact structure: JSON / table / numbered list / schema]

Prohibited: preambles, reasoning explanations, meta-comments.
Result only.
```

## What NOT to add

Do NOT add any of the following — they will degrade o4:
- State machine / status blocks
- Execution pipeline with phases
- "Think about this before answering"
- Expert swarm simulation
- Extended context blocks
- Ignition triggers / confirmation phrases
- Examples of reasoning

## Quality requirements

- Length: **2,000–6,000 characters** (this is critical — do not exceed)
- Only 4 elements: Role + Task + Constraints + Format
- No CoT, no reasoning instructions, no meta-layers
- Shorter is better for reasoning models
